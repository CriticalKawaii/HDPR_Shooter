## АННОТАЦИЯ

Дипломный проект на тему «Разработка компьютерной игры с применением XR-технологий» выполнен студентами специальности 09.02.07 Информационные системы и программирование.

Дипломный проект содержит приблизительно 70 страниц основного текста, 20 таблиц, 9 листингов в основной части, 7 приложений с полными листингами исходного кода; список использованных источников включает 25 наименований.

**Ключевые слова:** виртуальная реальность, XR-технологии, Unity, HDRP, XR Interaction Toolkit, C#, система оружия, стрелковый тренажёр, VR-взаимодействие, ScriptableObject, ConfigurableJoint, PBR-текстурирование.

В работе разработана компьютерная игра — интерактивный стрелковый VR-тренажёр на платформе Unity 6 с применением High Definition Render Pipeline (HDRP) и XR Interaction Toolkit 4.6.0. Реализованы: система оружия двух видов (штурмовая винтовка и пистолет) с физическими механиками затвора, магазина, гильз и отдачи VR-контроллера; поддержка платформ Oculus SDK и OpenXR; игровая логика стрелкового тренажёра с системой очков, мультипликаторами дальности и статистикой раунда. Проведено функциональное тестирование (14 тест-кейсов) и проверка производительности на устройстве Meta Quest 3 (90 FPS, GPU Frame Time 8–11 мс).

## ВВЕДЕНИЕ

Развитие технологий дополненной и виртуальной реальности за последнее десятилетие перешло из стадии экспериментов в стадию массового применения. Под аббревиатурой XR (Extended Reality, расширенная реальность) объединяются три направления: виртуальная реальность (VR), дополненная реальность (AR) и смешанная реальность (MR). Если ещё в 2016 году VR-гарнитуры первого поколения воспринимались как технологические диковинки с ограниченным кругом применения, то к 2024 году платформы Meta Quest 3, PlayStation VR2 и HTC VIVE Pro 2 стали полноценными коммерческими продуктами с многомиллионной аудиторией.

По данным аналитической компании IDC, совокупные расходы на технологии AR/VR в мире составили около 15,5 млрд долларов США в 2023 году и по прогнозам превысят 160 млрд долларов к 2026 году, демонстрируя среднегодовой темп роста свыше 40 %. Игровой сегмент при этом является крупнейшим драйвером этого рынка: VR-игры формируют основной поток загрузок и выручки на всех крупных платформах. Среди жанров особую популярность набрали игры в жанре Action/Shooter, которые органично сочетаются с VR-форматом: физические движения рук заменяют управление мышью, прицеливание становится интуитивным, а ощущение присутствия (presence) обеспечивает принципиально новый игровой опыт.

Параллельно с развлекательным применением XR-технологии активно проникают в сферу профессионального обучения. Стрелковые тренажёры на базе VR применяются в подготовке военнослужащих и сотрудников силовых структур, в спортивных стрелковых клубах и в образовательных целях. Ключевым преимуществом VR-тренажёра перед традиционным является безопасность, снижение стоимости расходных материалов и возможность объективной оценки результатов тренировки.

Несмотря на широкий выбор коммерческих VR-игр, ни одно из доступных приложений не сочетает одновременно фотореалистичное качество рендеринга на основе HDRP (High Definition Render Pipeline), поддержку актуальных OpenXR-платформ, детализированные физические механики оружия и систему объективной оценки точности стрельбы. Это определяет актуальность разработки собственного приложения, объединяющего перечисленные характеристики.

**Цель дипломного проекта** — разработать компьютерную игру с применением XR-технологий, представляющую собой интерактивный стрелковый тренажёр с реалистичными механиками оружия, системой мишеней и оценки результатов.

**Задачи проекта:**

1. Проанализировать предметную область XR-технологий и рынок существующих VR-приложений схожей тематики.
2. Выполнить сравнительный анализ аналогов и сформулировать требования к разрабатываемому приложению.
3. Разработать архитектуру приложения и обосновать выбор инструментальных средств.
4. Реализовать систему управления оружием: механизм стрельбы, перезарядки, затворной группы и отдачи (Студент 1).
5. Разработать 3D-ассеты, PBR-текстуры и HDRP-освещение игровой сцены (Студент 2).
6. Интегрировать XR-взаимодействие с применением XR Interaction Toolkit.
7. Разработать игровую логику: систему мишеней, подсчёт очков, таймер раунда.
8. Провести тестирование и разработать руководство по использованию.

**Объект исследования** — процесс разработки компьютерных игр с применением XR-технологий на платформе Unity.

**Предмет исследования** — методы и средства реализации игровых механик оружия и систем взаимодействия в VR-приложении.

**Методы исследования:**
- *Теоретические*: анализ научной и технической литературы, синтез и сравнение характеристик аналогов, моделирование архитектуры системы.
- *Эмпирические*: эксперимент при разработке и настройке физических механик, измерение частоты кадров и задержки ввода, функциональное тестирование по тест-кейсам.

**Практическая значимость** работы состоит в том, что разработанное приложение представляет собой готовый стрелковый тренажёр для VR-платформ (Oculus/Meta Quest, OpenXR-совместимые устройства), а также служит демонстрацией архитектурных решений для дальнейшего расширения арсенала оружия и игровых режимов.

**Распределение обязанностей между участниками:**

- *Студент 2* — визуальная составляющая: создание полигональных 3D-моделей в Blender, разработка PBR-текстур в GIMP (включая HDRP Mask Map), настройка HDRP-материалов и освещения сцены.

**Структура работы.** Дипломный проект состоит из введения, двух глав, заключения, списка литературы и приложений. В первой главе рассматривается предметная область, анализируются аналоги, формулируются требования, описываются архитектура и инструментальные средства. Вторая глава посвящена практической реализации: разработке сцены, системы оружия, XR-взаимодействия, игровой логики, пользовательского интерфейса и тестированию. ## ГЛАВА 1. ПРЕДПРОЕКТНОЕ ИССЛЕДОВАНИЕ ПРЕДМЕТНОЙ ОБЛАСТИ

### 1.1. Описание предметной области

**XR-технологии: классификация и применение**

XR (Extended Reality, расширенная реальность) — обобщающий термин для технологий, изменяющих восприятие пользователем окружающего пространства посредством цифровых средств. В зависимости от степени погружения в цифровую среду выделяют три основных направления.

**Виртуальная реальность (VR, Virtual Reality)** предполагает полное погружение пользователя в синтетическую трёхмерную среду. Взаимодействие с физическим миром при этом исключается: пользователь надевает шлем виртуальной реальности (HMD, Head-Mounted Display), который блокирует поле зрения и выводит стереоскопическое изображение. Современные VR-гарнитуры оснащены системами отслеживания положения головы (6DoF — шесть степеней свободы), контроллерами движения и, в ряде случаев, системой отслеживания рук без контроллеров. К ведущим платформам относятся: Meta Quest 3, PlayStation VR2, HTC VIVE Pro 2, Valve Index.

**Дополненная реальность (AR, Augmented Reality)** накладывает цифровые элементы поверх изображения реального мира, получаемого через камеру мобильного устройства или через прозрачный дисплей специальных очков. Пользователь сохраняет визуальный контакт с окружающей средой. Примеры платформ: Microsoft HoloLens 2, Apple Vision Pro, мобильные приложения на базе ARKit/ARCore.

**Смешанная реальность (MR, Mixed Reality)** объединяет элементы VR и AR: цифровые объекты воспринимаются как физически присутствующие в реальном пространстве и взаимодействуют с реальными поверхностями. Граница между AR и MR условна; Microsoft использует термин MR для HoloLens, тогда как Meta обозначает аналогичные возможности Quest 3 термином «сквозной вид» (Passthrough).

Разрабатываемое приложение относится к категории VR: оно полностью погружает пользователя в трёхмерную игровую среду и требует VR-гарнитуры с контроллерами движения.

**Рынок VR-технологий и игровой сегмент**

По данным аналитической компании IDC (International Data Corporation), совокупные глобальные расходы на технологии AR/VR составили около 15,5 млрд долларов в 2023 году. Прогноз на 2026 год — более 160 млрд долларов при среднегодовом темпе роста порядка 40 %. Основными факторами роста являются: доступность автономных гарнитур без привязки к ПК, улучшение разрешения и частоты обновления дисплеев, расширение библиотеки контента и снижение цены устройств.

Игровой сегмент стабильно формирует наибольшую долю загрузок и пользовательской аудитории на всех VR-платформах. Среди игровых жанров лидируют приложения категории Action и Shooter: интуитивное управление движением рук делает прицеливание и манипуляцию оружием максимально естественными в VR-среде.

Специфика VR-игр жанра Action/Shooter по сравнению с традиционными настольными играми:
- Управление оружием осуществляется реальными движениями рук в физическом пространстве, а не нажатием кнопок.
- Прицеливание производится путём наведения физического контроллера, а не перемещением курсора мыши.
- Перезарядка требует воспроизведения реальных жестов: извлечение магазина, вставка нового, передёргивание затвора.
- Эффект присутствия (presence) — ощущение физического нахождения в сцене — принципиально отличает VR-опыт от плоского экрана.

**Стрелковые тренажёры на базе VR**

Стрелковые тренажёры представляют собой специализированный класс VR-приложений, ориентированных не на нарратив, а на отработку навыков точной стрельбы с объективной оценкой результатов.

Области применения VR-стрелковых тренажёров:

- **Военная подготовка.** Армии ряда государств используют VR-системы для тренировки личного состава в условиях, максимально приближенных к боевым, без расхода боеприпасов и связанных рисков.
- **Правоохранительные органы.** Сценарные тренажёры позволяют отрабатывать тактические ситуации: выбор цели, принятие решений в условиях ограниченного времени.
- **Спортивная стрельба.** VR-тренажёры используются для межсезонных тренировок, недоступных на реальном стрельбище.
- **Развлекательные аркады.** VR-стрельбища предоставляют широкой аудитории опыт управления оружием в безопасных условиях.

Ключевые характеристики качественного стрелкового тренажёра:
- Физически достоверное поведение оружия (отдача, затвор, магазин).
- Система оценки результатов: точность, реакция, дистанция поражения.
- Динамические мишени с изменяющимся поведением.
- Высокое визуальное качество для поддержания эффекта присутствия.

Именно совокупность этих характеристик является целевым ориентиром разрабатываемого приложения и отличает его от чисто развлекательных VR-игр, рассмотренных в следующем разделе.

### 1.2. Анализ существующих аналогов

Для определения требований к разрабатываемому приложению выполнен сравнительный анализ четырёх наиболее релевантных существующих VR-приложений жанра Action/Shooter.

**Half-Life: Alyx (Valve Corporation, 2020)**

Half-Life: Alyx — флагманская VR-игра компании Valve, разработанная на проприетарном движке Source 2. Игра представляет собой полноценный сюжетный шутер от первого лица, ориентированный на PC VR-платформы (Valve Index, HTC VIVE, Oculus Rift). Механики оружия реализованы с высокой степенью реализма: пистолет оснащён интерактивным затвором, перезарядка требует физических жестов, патроны можно подбирать с поверхностей. Графическое качество обеспечивается собственным конвейером рендеринга Source 2, поддерживающим глобальное освещение и объёмный туман.

Вместе с тем приложение ориентировано исключительно на нарративный опыт: игрок движется по линейному сценарию без возможности выбора режима свободной стрельбы или тренировочного полигона. Система оценки точности и статистика попаданий отсутствуют. Использование проприетарного движка Source 2 исключает возможность изучения или переиспользования архитектурных решений в других проектах.

**Hot Dogs, Horseshoes & Hand Grenades (Rust Ltd, 2016)**

H3VR (Hot Dogs, Horseshoes & Hand Grenades) — VR-симулятор оружия, разработанный на платформе Unity. По концепции это приложение наиболее близко к разрабатываемому: в нём реализованы сотни видов оружия с детальной физической симуляцией всех механизмов, а один из режимов («Take and Hold») предполагает оборону позиции против волн противников на открытой площадке.

Однако проект был создан в 2016 году для устройств первого поколения — Oculus Rift CV1 и PlayStation VR первой версии, — которые к настоящему времени сняты с производства и официальной поддержки. Приложение использует Unity версии 5.x и Legacy Render Pipeline без поддержки HDRP. Визуальное качество, характерное для 2016 года, значительно уступает современным стандартам. SDK Oculus Runtime 1.x и устаревший Input Manager несовместимы с текущими XR Interaction Toolkit и OpenXR.

**Superhot VR (Superhot Team, 2016)**

Superhot VR — инновационное VR-приложение с уникальной механикой: время в игре движется лишь тогда, когда движется игрок. Это делает игру одновременно шутером и головоломкой. Визуальный стиль намеренно минималистичен — трёхцветная графика без текстур, без реалистичного освещения. Управление оружием упрощено: отсутствуют механика магазина, затвора и ограниченного боезапаса в традиционном смысле.

Приложение не ставит перед собой задачи физического реализма и не содержит инструментов оценки точности стрельбы. Используемый движок Unity не применяет HDRP. Проект предназначен для развлечения, а не для тренировки.

**Boneworks (Stress Level Zero, 2019)**

Boneworks — физически ориентированный VR Action, разработанный на Unity. Акцент приложения сделан на взаимодействии с окружением через физический движок: оружие имеет физическую массу, взаимодействует с поверхностями, а механика захвата и удержания предметов реализована через джойнты — подход, аналогичный применённому в данном проекте. Игра демонстрирует зрелую реализацию физических механик оружия на Unity и является важным техническим ориентиром.

Вместе с тем Boneworks ориентирован на сюжетное прохождение; специализированного тренировочного режима с оценкой результатов не предусмотрено. Визуальный конвейер — стандартный Universal Render Pipeline (URP), не HDRP. Приложение разработано под устаревший API SteamVR 1.x.

### 1.3. Требования к разрабатываемому приложению

### 1.4. Архитектура приложения и технологии решения задач

**Компонентная модель Unity**

Unity построена на паттерне «Сущность — Компонент» (Entity-Component): каждый объект сцены (GameObject) является контейнером, к которому прикрепляются компоненты (MonoBehaviour). Логика не концентрируется в монолитных иерархиях классов, а распределяется по изолированным компонентам, взаимодействующим через события и прямые ссылки. Это обеспечивает высокую модульность: добавление нового поведения не требует изменения существующего кода — достаточно создать новый компонент и прикрепить его к объекту.

В разрабатываемом приложении каждая единица оружия представлена набором специализированных компонентов, каждый из которых отвечает за одну функциональную область:

Таблица 5 — Компоненты системы оружия

| Компонент | Ответственность |
|---|---|
| `AsRifleShoot` / `PistolShoot` | Логика стрельбы: raycast, анимация, звук, вспышка |
| `BoltMechanics` / `PistolBoltMechanics` | Физика затвора, досылание патрона |
| `AsRifleGrab` / `PistolGrab` | Управление моделями рук при захвате/отпускании |
| `HybridRecoil` | Отдача: смещение и поворот VR-контроллера |
| `CasingSpawner` / `PistolCasingSpawner` | Выброс гильзы с физикой и рандомизацией |
| `Magazine` | Хранение количества патронов в магазине |
| `WeaponConfig` (ScriptableObject) | Конфигурация параметров оружия |
| `TriggerInput` | Получение события нажатия триггера из New Input System |
| `BoltController` | Универсальный контроллер затвора (новая архитектура) |
| `BarrelExplode` | Взрыв разрушаемого объекта при попадании |

**Схема взаимодействия подсистем**

Поток данных при выстреле выглядит следующим образом:

```
XR-контроллер (триггер)
        │
        ▼
  TriggerInput
  (New Input System)
        │  TriggerPress()
        ▼
  AsRifleShoot
  ├── Проверка: болт закрыт? патрон в патроннике?
  ├── gunanim.Play("Shoot")
  ├── AudioSource.PlayOneShot(bangClip)
  ├── Physics.Raycast → попадание?
  │       ├── SpawnImpact()     — визуальный след пробоины
  │       ├── Rigidbody.AddForce — физический импульс объекту
  │       └── BarrelExplode.Explode() — если тег "Barrel"
  ├── CasingSpawner.Spawn()    — физическая гильза
  ├── HybridRecoil.Fire()      — смещение контроллера
  └── Coroutine: FlashLight()  — вспышка дула (0,05 с)
```

При попадании в мишень событие передаётся в GameManager, который обновляет счёт с учётом типа попадания (тело/голова) и множителя дальности.

**ScriptableObject-конфигурация**

Параметры каждого вида оружия вынесены в ScriptableObject-актив `WeaponConfig`. ScriptableObject — специальный тип Unity-объекта, хранящийся как `.asset`-файл в проекте, не привязанный к конкретной сцене. Преимущества этого подхода:

- Изменение параметров (скорострельность, дальность, характеристики отдачи) производится в редакторе Unity без перекомпиляции проекта.
- Один и тот же `.asset` может назначаться нескольким префабам: все экземпляры автоматически получают изменения.
- Конфигурации разных видов оружия хранятся отдельными файлами, что упрощает управление версиями.

**Архитектура игровой логики**

GameManager реализует конечный автомат с состояниями: `Idle → Countdown → RoundActive → RoundEnd`. Переходы между состояниями:

- `Idle → Countdown`: нажатие кнопки в сцене
- `Countdown → RoundActive`: истечение отсчёта (3 секунды)
- `RoundActive → RoundEnd`: истечение таймера раунда ИЛИ счёт < 0

В состоянии `RoundActive` каждый кадр обрабатываются: движение мишеней, проверка пересечения границы, обновление UI.

**Вывод по разделу.** Компонентная архитектура Unity в сочетании со ScriptableObject-конфигурацией и событийной моделью взаимодействия обеспечивает модульность, расширяемость и чёткое разделение ответственности между подсистемами приложения.

### 1.5. Обоснование выбора инструментальных средств

**Unity 6000.4.3f1**

Unity — ведущий игровой движок для разработки VR-приложений. По данным самой компании, более 60 % VR/AR-приложений на крупнейших платформах создано с применением Unity. Ключевые основания для выбора:

- Официальный пакет XR Interaction Toolkit обеспечивает готовую систему VR-взаимодействий без необходимости реализации с нуля.
- HDRP (High Definition Render Pipeline) входит в стандартную экосистему Unity и не требует сторонних лицензий.
- Поддержка Oculus и OpenXR реализована через единый XR Management API.
- Язык C# с поддержкой .NET обеспечивает строгую типизацию и развитые инструменты отладки.

Альтернативой мог служить Unreal Engine 5 с системой Lumen и Nanite. Однако Unreal требует значительно больше оперативной и видеопамяти в VR-сценах, кривая освоения движка существенно выше, а XR-экосистема менее зрелая по сравнению с Unity XR Interaction Toolkit.

Таблица 6 — Сравнение Unity и Unreal Engine 5 для VR-разработки

| Критерий | Unity 6 + HDRP | Unreal Engine 5 |
|---|---|---|
| XR-экосистема | XRI 4.6 (зрелая) | OpenXR-плагин (развивающийся) |
| Язык программирования | C# | C++ / Blueprint |
| Потребление памяти в VR | Умеренное | Высокое |
| Время компиляции | Быстрое | Медленное (Shader Compilation) |
| Документация по VR | Обширная | Ограниченная |
| Порог вхождения | Средний | Высокий |

**HDRP 17.4.0 (High Definition Render Pipeline)**

HDRP — конвейер рендеринга Unity для высококачественной и фотореалистичной графики. Применяется в данном проекте для:

- Физически корректного освещения (PBR) на основе материалов Lit с поддержкой Mask Map.
- Запечённого (baked) глобального освещения с цветовыми и направленными лайтмапами.
- Пост-обработки: Screen Space Ambient Occlusion (SSAO), Bloom, Tone Mapping, Vignette.
- Поддержки Reflection Probe для глянцевых поверхностей.

HDRP Mask Map — специальный формат текстуры, объединяющий четыре карты в одном файле: R-канал — Metallic, G-канал — Ambient Occlusion, B-канал — Detail Mask, A-канал — Smoothness. Это сокращает количество текстурных сэмплеров и увеличивает производительность.

**XR Interaction Toolkit 4.6.0**

Официальный фреймворк Unity для разработки XR-приложений. Предоставляет готовые компоненты:

- `XRGrabInteractable` — компонент захвата физических объектов рукой контроллера.
- `XRSocketInteractor` — «гнездо» для вставки объектов, применяемое для системы магазинов.
- `XRSimpleInteractable` — упрощённое взаимодействие, используемое для затвора.
- `XR Origin` — стандартная иерархия VR-игрока с камерой и контроллерами.

**New Input System 1.19.0**

Современная система ввода Unity, пришедшая на смену устаревшему Legacy Input Manager. Ключевые преимущества в контексте VR-разработки:

- Привязка действий к физическим кнопкам контроллера через графический редактор Input Actions.
- Асинхронная модель событий (`performed`, `canceled`) вместо опроса состояния каждый кадр.
- Полная независимость от платформы: один и тот же код работает на Oculus Touch и OpenXR-контроллерах.

**Visual Studio 2022**

Интегрированная среда разработки Microsoft с расширением для Unity. Обеспечивает: полное автодополнение кода C# с учётом Unity API, встроенный отладчик с возможностью точек останова прямо в работающей сцене Unity, навигацию по коду (Go to Definition, Find All References).

**GIMP 2.10**

Свободный растровый редактор с открытым исходным кодом. Применялся Студентом 2 для:

- Создания PBR-текстур: карт Albedo, Normal Map, Roughness.
- Составления HDRP Mask Map: ручное разнесение монохромных изображений по цветовым каналам (R/G/B/A) с последующим сохранением в формате PNG без сжатия.
- Ретуши и доработки текстур, полученных из открытых источников (CC0-лицензия).

GIMP выбран как бесплатная альтернатива Adobe Photoshop с полноценной поддержкой работы с каналами, необходимой для создания Mask Map.

**Blender 4.x**

Свободная 3D-платформа для полного цикла создания трёхмерного контента. Применялась Студентом 2 для:

- Полигонального моделирования объектов окружения (мебель, элементы тира).
- Построения UV-развёрток (UV Unwrapping) для корректного наложения текстур без искажений.
- Экспорта моделей в формат FBX для последующего импорта в Unity с сохранением UV-координат и нормалей.

**Вывод по разделу.** Выбранный стек инструментов — Unity 6 + HDRP + XRI + New Input System + Visual Studio + GIMP + Blender — обеспечивает полный цикл разработки VR-приложения: от программирования игровой логики до создания визуальных ресурсов. Все инструменты либо являются отраслевым стандартом (Unity, Visual Studio), либо предоставляют функциональность, сопоставимую с коммерческими аналогами, на условиях бесплатной лицензии (Blender, GIMP).

**HDRP-освещение**

Освещение сцены реализовано с применением комбинированного подхода: статические объекты окружения используют запечённое глобальное освещение, динамические объекты (оружие, мишени, руки) освещаются зондами отражений и динамическими источниками.

В папке `Assets/Scenes/ShootingRange/` хранятся результаты запекания освещения:
- `Lightmap-0_comp_light.exr` и `Lightmap-1_comp_light.exr` — два лайтмапа яркости в формате HDR, покрывающие всю поверхность сцены;
- `Lightmap-0_comp_dir.png` и `Lightmap-1_comp_dir.png` — направленные лайтмапы, кодирующие преобладающее направление падения света для корректного отображения нормалей на статичных поверхностях;
- `ReflectionProbe-0.exr` — кубическая карта отражений в формате HDR, запечённая из центра сцены; используется HDRP-материалами с ненулевой гладкостью (Smoothness > 0) для корректных глянцевых отражений;
- `LightingData.asset` — база данных запечённого освещения Unity, включающая данные зондов освещения (Light Probes) для динамических объектов.

HDRP-профиль сцены выбирается из трёх заготовленных вариантов качества: `HDRP Performant.asset` (для слабых устройств), `HDRP Balanced.asset` (рабочий профиль) и `HDRP High Fidelity.asset` (для скриншотов и демонстрации). Активным является профиль `Balanced`, обеспечивающий баланс между визуальным качеством и производительностью.

**Global Volume и пост-обработка**

Объект `Global Volume` со связанным профилем `Global Volume Profile.asset` управляет пост-обработкой всей сцены. Профиль включает следующие эффекты:

- **Sky and Fog Settings** (`SkyandFogSettingsProfile.asset`) — атмосферный туман, создающий ощущение закрытого пространства и скрывающий дальние границы сцены;
- **Screen Space Ambient Occlusion (SSAO)** — экранное ambient occlusion, затемняющее углы и щели для усиления объёмности геометрии;
- **Bloom** — эффект свечения вокруг ярких источников (дульная вспышка, индикаторы);
- **Tone Mapping** — кривая тональной компрессии HDR-изображения в диапазон экрана;
- **Vignette** — лёгкое затемнение краёв кадра для усиления фокуса на центре сцены.

**Создание 3D-ассетов и текстурирование (Студент 2)**

Часть объектов окружения и мишени разработаны Студентом 2 с применением Blender и GIMP. Рабочий процесс состоял из следующих этапов.

*Этап 1. Моделирование в Blender.* Полигональные модели создавались с контролем количества полигонов: для мишени (FBX-файлы `target.fbx`, `target 1.fbx`) применялась низкополигональная (low-poly) геометрия, достаточная для чёткой визуальной читаемости в VR. После завершения моделирования выполнялась UV-развёртка (UV Unwrap): каждая грань меша проецировалась на плоскость UV-пространства без перекрытий, что обеспечивает корректное и однородное наложение текстур.

*Этап 2. Текстурирование в GIMP.* Для каждого объекта создавался набор PBR-текстур:
- **Albedo** (Base Color) — цветовая текстура без информации об освещении;
- **Normal Map** — карта нормалей в тангентном пространстве для имитации рельефа поверхности без увеличения полигонажа;
- **Mask Map** — составная текстура в формате HDRP.

*Этап 3. Составление HDRP Mask Map в GIMP.* HDRP-материал типа Lit принимает Mask Map — единую текстуру, объединяющую четыре канала:

Таблица 11 — Структура HDRP Mask Map

| Канал | Данные | Описание |
|---|---|---|
| R (красный) | Metallic | Металличность поверхности (0 — диэлектрик, 1 — металл) |
| G (зелёный) | Ambient Occlusion | Затенение в труднодоступных местах поверхности |
| B (синий) | Detail Mask | Маска для наложения Detail Map (дополнительной детализации) |
| A (альфа) | Smoothness | Гладкость/шероховатость поверхности |

В GIMP процедура сборки Mask Map выполнялась следующим образом: каждая из четырёх монохромных карт открывалась отдельно, затем через диалог «Использовать как канал» (Channels) каждая копировалась в соответствующий цветовой канал нового RGBA-изображения. Финальный файл сохранялся в формате PNG без сжатия для сохранения точности значений.

Аналогичная структура текстур реализована для моделей рук игрока (папка `Assets/SimpleHands/Textures/`): `B-Base.png` / `W-Base.png` — Albedo, `B-AO.png` / `W-AO.png` — Ambient Occlusion, `B-M.png` / `W-M.png` — Metallic/Mask для чёрной и белой моделей рук соответственно.

*Этап 4. Настройка HDRP-материалов в Unity.* После импорта текстур создавались материалы типа `HDRP/Lit`. В инспекторе материала назначались: Albedo в слот Base Map, Normal Map в слот Normal Map, составная Mask Map в слот Mask Map. Значения Surface Type устанавливались в `Opaque`, Rendering Pass — в `Forward`.

**Система стрельбы**

Стрельба реализована методом хитскан (hitscan): при нажатии триггера мгновенно пускается луч `Physics.Raycast`. Это технически проще и производительнее физических снарядов, а для дистанций стрелкового тренажёра — достаточно точно.

Центральный метод стрельбы `PerformShoot()` компонента `AsRifleShoot` выполняет следующую последовательность:

Листинг 2 — AsRifleShoot.cs, метод PerformShoot

```csharp
private void PerformShoot()
{
    if (!ammoInRifle)
    {
        gunanim?.Play("ShootEmpty");
        return;
    }

    gunanim?.Play("Shoot");
    bang?.PlayOneShot(bangClip);
    ApplyRecoil();
    StartCoroutine(SpawnWithDelay()); // гильза через 1 кадр

    Vector3 dir = shootPoint.forward + new Vector3(
        Random.Range(-0.01f, 0.01f),
        Random.Range(-0.01f, 0.01f), 0f);

    if (Physics.Raycast(shootPoint.position, dir.normalized,
                        out RaycastHit hit, range))
    {
        SpawnImpact(hit);
        if (hit.rigidbody != null)
            hit.rigidbody.AddForceAtPosition(
                dir * 10f, hit.point, ForceMode.Impulse);
        if (hit.collider.CompareTag("Barrel"))
            hit.collider.GetComponent<BarrelExplode>()?.Explode();
    }

    if (currentAmmo > 0) { currentAmmo--; magazineScript.Ammo = currentAmmo; }
    else                   ammoInRifle = false;

    StartCoroutine(FlashLight());
}
```

Вектор направления луча случайно отклоняется в пределах ±0,01 по осям X и Y, имитируя естественный разброс. При обнаружении попадания создаётся визуальный след пробоины (`SpawnImpact`), объект получает физический импульс, а при теге `"Barrel"` вызывается взрыв. Вспышка дула (`FlashLight`) включает источник света `muzzleLight` на 50 мс.

Тип стрельбы (одиночный / автоматический) управляется полем `fireMode` конфигурации. При автоматическом режиме флаг `triggerPressed` остаётся установленным пока удерживается триггер, и выстрелы производятся с интервалом `1f / fireRate` секунд.

**Механика затвора**

Затвор оружия реализован через компонент Unity `ConfigurableJoint`, который ограничивает движение объекта: оси X и Y зафиксированы, вдоль оси Z разрешено линейное перемещение. Это позволяет игроку физически потянуть рукоять затвора в VR-пространстве.

Листинг 3 — BoltMechanics.cs, методы управления и автовозврата

```csharp
public void Grabbed()
{
    Release    = false;
    Based      = false;    // затвор открыт
    isFreezed  = false;    // снять фиксацию оси Z
    GunAnim.enabled = false; // отключить аниматор на время физики
}

public void Released()
{
    Release = true;        // разрешить автовозврат
}

void Update()
{
    if (Release && !Based)
    {
        Vector3 local =
            reference.InverseTransformPoint(transform.position);
        // порог закрытого положения: 0.056 м / -0.054 м
        if (local.z <= 0.056f && local.z >= -0.054f)
        {
            trans.localPosition =
                new Vector3(0.05256491f, 0.01729099f, 0);
            Release = Based = true;
            isFreezed = true;
            GunAnim.enabled = true;
            Asrfl.TryToReload(); // досылание патрона
        }
    }

```

Флаг `Based` означает «затвор закрыт и зафиксирован»: только при `Based == true` компонент стрельбы разрешает произвести выстрел (`IsClosed()` возвращает `Based`). При отпускании рукояти затвор под действием пружины возвращается в закрытое положение — отслеживается по пороговому значению локальной Z-координаты (0,056 м). После достижения порога затвор привязывается к закрытой позиции, аниматор включается, и вызывается `TryToReload()` для досылания патрона.

`LateUpdate` фиксирует оси X и Y в сохранённых значениях каждый кадр, не допуская нежелательного дрейфа по этим осям под влиянием физики.

**Физика гильз**

`CasingSpawner` создаёт физический объект-гильзу при каждом выстреле. Задержка в один кадр (`yield return null`) синхронизирует спавн с анимацией выброса затвора.

Листинг 5 — CasingSpawner.cs, метод Spawn (ключевая часть)

```csharp
public void Spawn()
{
    Transform target = spawnPoint != null ? spawnPoint : transform;

    Quaternion rot = target.rotation
        * Quaternion.Euler(rotationOffset)
        * Quaternion.Euler(-90f, 0f, 0f); // ориентация гильзы

    GameObject obj = Instantiate(prefab, target.position, rot);

    Rigidbody rb = obj.GetComponent<Rigidbody>()
                   ?? obj.AddComponent<Rigidbody>();

    Vector3 worldDir = target.TransformDirection(
        RandomizeDirection(impulseDirection).normalized);

    rb.AddForce(worldDir * RandomizeForce(impulseForce),
                ForceMode.Impulse);

    Destroy(obj, lifeTime); // удалить через 60 с
}

private float RandomizeForce(float f)
{
    float p = forceRandomPercent / 100f;
    return f * Random.Range(1f - p, 1f + p); // ±20% по умолчанию
}
```

Рандомизация силы (±20 %) и направления (±15 %) обеспечивает естественное, непредсказуемое поведение каждой гильзы. Поворот −90° по оси X корректирует ориентацию модели гильзы относительно точки выброса. Автоматическое уничтожение через 60 секунд предотвращает накопление объектов в сцене.

**Система отдачи**

`HybridRecoil` реализует обратную связь непосредственно на трансформ VR-контроллера: при выстреле контроллер смещается назад и поворачивается вверх, затем плавно возвращается в исходное положение за два кадрово-независимых этапа.

Листинг 6 — HybridRecoil.cs, метод Fire и фаза Update

```csharp
public void Fire()
{
    if (isRecoiling || grabInteractable == null) return;

    var interactors = grabInteractable.interactorsSelecting;
    if (interactors.Count == 0) return;

    hands = new HandData[interactors.Count];
    for (int i = 0; i < interactors.Count; i++)
    {
        var ctrl = interactors[i].transform;
        hands[i] = new HandData {
            controller       = ctrl,
            originalPosition = ctrl.position,
            originalRotation = ctrl.rotation
        };
    }
    isRecoiling = true;
    recoilTimer = 0f;
}

// Фрагмент Update — фаза 1: отдача
if (recoilTimer <= recoilDuration)
{
    float t = recoilTimer / recoilDuration;
    Vector3 offset = -ctrl.forward * recoilDistance
                   +  ctrl.up     * (recoilDistance * 0.5f);
    ctrl.position = hands[i].originalPosition + offset * (1f - t);
    ctrl.rotation = hands[i].originalRotation
                  * Quaternion.Euler(-recoilAngle * (1f - t), 0, 0);
}
```

Параметры по умолчанию: смещение — 0,15 м, угол подброса — 8°, длительность отдачи — 0,1 с, скорость возврата — 5. Система поддерживает двуручный хват: при захвате оружия двумя руками отдача применяется к обоим контроллерам одновременно.

```csharp
        if (rightHandOnWeapon.activeSelf) // правая уже держит
        {
            leftHandModel.SetActive(showPlayerHand);
            leftSecondHand.SetActive(!showPlayerHand); // вторичный хват
        }
        else
        {
            leftHandModel.SetActive(showPlayerHand);
            leftHandOnWeapon.SetActive(!showPlayerHand);
        }
    }
    else if (t.CompareTag("RightHand"))
    {
        if (leftHandOnWeapon.activeSelf) // левая уже держит
        {
            rightHandModel.SetActive(showPlayerHand);
            rightSecondHand.SetActive(!showPlayerHand);
        }
        else
        {
            rightHandModel.SetActive(showPlayerHand);
            rightHandOnWeapon.SetActive(!showPlayerHand);
        }
    }
}
```

Идентификация руки производится по тегам `"LeftHand"` / `"RightHand"`, назначенным GameObject-объектам контроллеров в иерархии XR Origin.

**Пистолет**

Система пистолета (`PistolShoot`, `PistolBoltMechanics`, `PistolGrab`, `PistolCasingSpawner`) реализована по той же архитектуре, что и система винтовки. Ключевые отличия:

- `PistolShoot` не имеет режима Auto — только Single (одиночный огонь при каждом нажатии триггера);
- `PistolCasingSpawner.PistolCasSpawn(float delay)` вызывает Spawn с задержкой через `Invoke`, а не через корутину;
- `PistolGrab` не поддерживает вторичный хват (пистолет держится одной рукой), поэтому поля `leftSecondHand` и `rightSecondHand` отсутствуют;
- параметры пистолета — отдельный файл `WeaponConfig`, настроенный под более высокую скорострельность и меньшую дальность.

**Разрушаемые объекты**

Взрывные бочки размещены в сцене как декоративные объекты с компонентом `BarrelExplode` и тегом `"Barrel"`. При попадании пули из `AsRifleShoot.PerformShoot()` вызывается:

Листинг 8 — BarrelExplode.cs

```csharp
public class BarrelExplode : MonoBehaviour
{

    public GameObject barrel;       // меш бочки
```

**Вывод по разделу.** Реализована полная система оружия для двух единиц (винтовка и пистолет), включающая: хитскан-стрельбу с разбросом, физический затвор на ConfigurableJoint, систему магазина через XRSocketInteractor, физику гильз с рандомизацией, двухфазную отдачу VR-контроллера, динамическую смену моделей рук и разрушаемые объекты. Все требования ФТ-02 — ФТ-10 выполнены.

**Поддержка XR-платформ**

Проект настроен на поддержку двух XR-провайдеров через пакет `XR Management`. Конфигурация хранится в:

- `Assets/XR/Settings/OpenXR Package Settings.asset` — настройки OpenXR (HTC VIVE, Valve Index, WMR-устройства).

Оба провайдера активированы в `XRGeneralSettingsPerBuildTarget.asset` и выбираются автоматически в зависимости от целевой платформы сборки: Android-сборка использует Oculus SDK, PC-сборка — OpenXR. Это обеспечивает выполнение требования НФТ-02 без дублирования логики взаимодействия.

**Обработка ввода с триггера**

Нажатие триггера VR-контроллера обрабатывается компонентом `TriggerInput`, использующим New Input System. Компонент подписывается на `InputActionReference` — ссылку на действие, настроенное в редакторе Input Actions Asset.

Листинг 9 — TriggerInput.cs

```csharp
public class TriggerInput : MonoBehaviour
{
    public InputActionReference triggerAction;
    private WeaponController    _weapon;
    private XRGrabInteractable  _grab;

    void Awake()
    {
        _weapon = GetComponent<WeaponController>();
        _grab   = GetComponent<XRGrabInteractable>();
    }

    void OnEnable()
    {
        triggerAction.action.performed += OnTriggerPerformed;
        triggerAction.action.canceled  += OnTriggerCanceled;
        triggerAction.action.Enable();
    }

    void OnDisable()
    {
        triggerAction.action.performed -= OnTriggerPerformed;
        triggerAction.action.canceled  -= OnTriggerCanceled;
    }

    private void OnTriggerPerformed(InputAction.CallbackContext _)
    {
        if (_grab.isSelected) _weapon.TriggerPress();
    }

    private void OnTriggerCanceled(InputAction.CallbackContext _)
    {
        _weapon.TriggerRelease();
    }
}
```

Проверка `_grab.isSelected` исключает выстрел в ситуации, когда триггер нажат, но оружие не находится в руке игрока (например, при взаимодействии с другими объектами). Подписка через события (`performed` / `canceled`) вместо опроса в `Update` снижает накладные расходы и обеспечивает мгновенную реакцию на ввод.

**Захват оружия (XRGrabInteractable)**

Каждое оружие оснащено компонентом `XRGrabInteractable`. При захвате XR Interaction Toolkit автоматически:
- прикрепляет объект оружия к трансформу контроллера;
- переключает Rigidbody в кинематический режим для предотвращения физических конфликтов;
- генерирует события `selectEntered` / `selectExited`, на которые подписаны `AsRifleGrab` и `AsRifleShoot`.

Параметр `Attach Transform` на `XRGrabInteractable` задаёт точку и ориентацию, в которой оружие фиксируется относительно контроллера — настраивается отдельно для рукояти каждого вида оружия.

**Физическая кнопка запуска раунда**

Для запуска раунда в сцене размещена физическая кнопка из набора XR Interaction Toolkit Samples — `XR Button Interactable`. Она реагирует на нажатие кончиком контроллера или рукой игрока, срабатывая при достижении порогового значения хода кнопки. Событие `OnPress` передаётся в GameManager.

Такое решение предпочтительнее UI-кнопки: оно сохраняет иммерсию (погружение), поскольку игрок совершает реальное физическое нажатие, а не выбирает пункт меню.

**Модели рук**

В проекте используются две пары моделей рук из пакета `SimpleHands`:
- **BlackHand** (`B-Base.png`, `B-AO.png`, `B-M.png`) — тёмная кожа;
- **WhiteHand** (`W-Base.png`, `W-AO.png`, `W-M.png`) — светлая кожа.

Каждая пара снабжена PBR-текстурами: Albedo, Ambient Occlusion и Mask Map. Модели динамически показываются и скрываются компонентами `AsRifleGrab` / `PistolGrab` при захвате и отпускании оружия (см. раздел 2.2).

**Вывод по разделу.** Реализовано полноценное XR-взаимодействие: иерархия XR Origin с отслеживанием головы и рук, одновременная поддержка Oculus SDK и OpenXR через XR Management, событийная обработка триггера через New Input System, захват оружия через XRGrabInteractable, физическая кнопка запуска раунда. Требования НФТ-02 и ФТ-01, ФТ-02, ФТ-11 выполнены.

### 2.4. Реализация игровой логики

Игровая логика приложения управляется компонентом `GameManager` — центральным синглтоном сцены, координирующим все игровые подсистемы: таймер, спавн мишеней, подсчёт очков и отображение результатов.

**Конечный автомат раунда**

`GameManager` реализован на основе конечного автомата с четырьмя состояниями:

Таблица 13 — Состояния конечного автомата GameManager

| Состояние | Описание | Переход |
|---|---|---|
| `Idle` | Ожидание. Граница скрыта, мишени отсутствуют | Нажатие кнопки → `Countdown` |
| `Countdown` | Обратный отсчёт 3–2–1, UI отображает цифры | Отсчёт завершён → `RoundActive` |
| `RoundActive` | Активный раунд: таймер идёт, мишени спавнятся | Таймер истёк ИЛИ счёт < 0 → `RoundEnd` |
| `RoundEnd` | Раунд завершён, UI показывает статистику | Кнопка → `Countdown` (новый раунд) |

При переходе `Countdown → RoundActive` в сцене появляется красная граница (`BoundaryLine`), обозначающая предельную дистанцию, до которой мишень не должна дойти. Одновременно в очередь спавна добавляется первая партия мишеней и один магазин.

**Спавн и движение мишеней**

Мишени создаются в случайных точках зоны спавна, расположенной на дальнем конце полигона. Начальная поза мишени — лежачая (поворот 90° относительно вертикальной оси); при спавне запускается анимация подъёма в вертикальное положение, имитирующая классическую мишень тира.

После перехода в вертикальное положение мишень начинает двигаться по прямой в сторону игрока с постоянной скоростью. Каждый кадр в состоянии `RoundActive` GameManager проверяет дистанцию от мишени до красной границы:

```csharp
// Псевдокод логики движения и проверки границы
void Update()
{
    if (state != RoundActive) return;

    foreach (var target in activeTargets)
    {
        target.transform.position +=
            Vector3.forward * targetSpeed * Time.deltaTime;

        if (target.transform.position.z <= boundaryZ)
        {
            AddScore(-300);         // штраф за пропущенную мишень
            stats.missedTargets++;
            Destroy(target);
            SpawnNextTarget();
        }
    }
}
```

При пересечении границы мишень уничтожается, счёт уменьшается на 300 очков, статистика пропущенных мишеней увеличивается, а на её место спавнится новая.

**Система подсчёта очков**

При попадании пули в мишень `AsRifleShoot` определяет тип попадания по тегу коллайдера и сообщает результат в GameManager. Мишень имеет два коллайдера: основной (тело) с тегом `"Target"` и дочерний (голова) с тегом `"TargetHead"`.

Таблица 14 — Начисление очков

| Событие | Базовые очки | Условие |
|---|---|---|
| Попадание в тело мишени | +50 | Тег `"Target"` |
| Попадание в голову (хэдшот) | +100 | Тег `"TargetHead"` |
| Мишень пересекла границу | −300 | Позиция ≤ `boundaryZ` |
| Подбор дополнительного магазина | −50 | Событие подбора |

Начисление очков производится с учётом мультипликатора дальности, вычисляемого в момент попадания:

```csharp
float GetDistanceMultiplier(Vector3 hitPoint)
{
    float dist = Vector3.Distance(playerPosition, hitPoint);
    if (dist > 40f) return 2.0f;
    if (dist > 25f) return 1.5f;
    return 1.0f;
}

void RegisterHit(string tag, Vector3 hitPoint)
{
    float multiplier = GetDistanceMultiplier(hitPoint);
    int   baseScore  = tag == "TargetHead" ? 100 : 50;

    AddScore(Mathf.RoundToInt(baseScore * multiplier));

    if (tag == "TargetHead") stats.headshots++;
    else                      stats.hits++;
}
```

Мультипликатор стимулирует игрока вести огонь по дальним мишеням, не подпуская их к себе: хэдшот на дистанции свыше 40 м даёт 200 очков, тогда как попадание в тело вплотную — лишь 50.

**Управление боезапасом в раунде**

В начале раунда на столе возле игрока спавнится один магазин. Когда игрок его подбирает (событие `XRGrabInteractable.selectEntered`), GameManager:
1. Вычитает 50 очков как штраф за использование дополнительного боезапаса.
2. Планирует спавн следующего магазина через фиксированный интервал.

Механика создаёт ресурсное давление: бесконтрольный расход патронов суммарно снижает итоговый счёт. Оптимальная стратегия — точная стрельба с максимальным числом хэдшотов и минимальным количеством смен магазина.

**Условия завершения раунда**

Раунд завершается при выполнении одного из двух условий, проверяемых каждый кадр:

```csharp
void CheckRoundEndConditions()
{
    if (roundTimer <= 0f || currentScore < 0)
        TransitionTo(RoundEnd);
}
```

Завершение по таймеру (стандартный сценарий) обеспечивает фиксированную продолжительность раунда. Завершение по отрицательному счёту служит «страховкой»: если игрок систематически пропускает мишени, раунд досрочно останавливается, не давая счёту уходить в произвольно глубокий минус.

**VFX очков**

При попадании в мишень рядом с точкой контакта создаётся всплывающий TextMeshPro-объект, отображающий набранные очки (`+50`, `+100`, `+200` и т. д.) с учётом мультипликатора. Объект анимируется: плавно поднимается вверх и исчезает через 1,5 секунды. Это обеспечивает мгновенную визуальную обратную связь без необходимости смотреть на UI-панель.

**Статистика раунда**

### 2.5. Разработка пользовательского интерфейса

**Подход к UI в VR**

Традиционный экранный (Screen Space) UI в VR непригоден: он накладывается поверх всей сцены на фиксированном расстоянии и создаёт конфликт между стереоскопической глубиной сцены и плоским изображением интерфейса, что вызывает дискомфорт. В разрабатываемом приложении все интерфейсные элементы размещены в режиме **World Space Canvas**: Canvas-объект существует в трёхмерном пространстве сцены как обычный 3D-объект с фиксированными координатами, размером и ориентацией. Игрок воспринимает его как физически присутствующий дисплей — аналог монитора на стойке тира.

**Элементы интерфейса**

UI-дисплей расположен на столе рядом с оружием и состоит из нескольких зон:

Таблица 16 — Элементы пользовательского интерфейса

| Элемент | Тип | Состояние отображения | Описание |
|---|---|---|---|
| Кнопка «Старт» | XR Button Interactable | `Idle`, `RoundEnd` | Физическая кнопка запуска раунда |
| Обратный отсчёт | TextMeshPro | `Countdown` | Цифры 3 → 2 → 1 → «Старт!» |
| Таймер раунда | TextMeshPro | `RoundActive` | Оставшееся время в формате М:СС |
| Счёт | TextMeshPro | `RoundActive`, `RoundEnd` | Текущий счёт (обновляется в реальном времени) |
| Попадания | TextMeshPro | `RoundActive`, `RoundEnd` | Количество попаданий в тело |
| Хэдшоты | TextMeshPro | `RoundActive`, `RoundEnd` | Количество попаданий в голову |
| Промахи | TextMeshPro | `RoundActive`, `RoundEnd` | Количество выстрелов без попадания |
| Пропущенные | TextMeshPro | `RoundActive`, `RoundEnd` | Мишени, достигшие границы |
| Итоговая таблица | Canvas Group | `RoundEnd` | Полная статистика завершённого раунда |

Видимость групп элементов управляется компонентом `Canvas Group` (свойство `alpha`): в состоянии `Idle` отображается только кнопка старта; в `Countdown` — только обратный отсчёт; в `RoundActive` — таймер и текущие показатели; в `RoundEnd` — итоговая таблица.

**TextMeshPro**

Для всего текстового контента используется пакет TextMeshPro, обеспечивающий чёткое отображение текста в VR без артефактов растеризации, характерных для стандартного Unity UI Text. TextMeshPro применяет технологию Signed Distance Field (SDF): текст остаётся чётким при любом масштабировании и угле обзора, что критично при размещении UI в мировом пространстве.

**VFX-индикаторы очков**

При каждом попадании в мишень GameManager создаёт временный объект с компонентом `TextMeshPro` рядом с точкой попадания. Объект отображает итоговые очки с учётом мультипликатора (`+50`, `+100`, `+200` и т. д.), плавно поднимается вверх на 0,5 м и исчезает через 1,5 секунды. Анимация реализована через `LeanTween` или стандартную корутину с интерполяцией:

```csharp
IEnumerator AnimateScorePopup(GameObject popup, int score)
{
    var tmp  = popup.GetComponent<TextMeshPro>();
    tmp.text = score > 0 ? $"+{score}" : $"{score}";

    float elapsed = 0f;
    Vector3 startPos = popup.transform.position;

    while (elapsed < 1.5f)
    {
        elapsed += Time.deltaTime;
        float t = elapsed / 1.5f;
        popup.transform.position = startPos + Vector3.up * (0.5f * t);
        tmp.alpha = 1f - t;          // плавное исчезновение
        yield return null;
    }
    Destroy(popup);
}
```

Цвет текста зависит от знака очков: зелёный для положительных, красный для отрицательных, золотой для хэдшотов.

**Физическая кнопка**

Объект кнопки реализован на базе `XR Button Interactable` из набора XR Interaction Toolkit Samples. Кнопка имеет физический ход (ConfigurableJoint, ограниченный по вертикальной оси), срабатывает при достижении порогового значения нажатия и отправляет UnityEvent в GameManager. Такой подход поддерживает эффект присутствия: игрок действительно «нажимает» кнопку рукой, а не взаимодействует с плоским меню.

**Вывод по разделу.** Пользовательский интерфейс реализован в режиме World Space Canvas, что обеспечивает корректное отображение в VR. Применение TextMeshPro гарантирует чёткость текста при любых углах обзора. Динамическое управление видимостью через Canvas Group соответствует текущему состоянию конечного автомата. Требования ФТ-12, ФТ-13, ФТ-18 выполнены.

### 2.7. Руководство по использованию

**Руководство пользователя**

Данный раздел описывает порядок работы с приложением для конечного пользователя — игрока в VR-гарнитуре.

*Системные требования:*
- VR-гарнитура: Meta Quest 2 / Meta Quest 3 (рекомендуется) или OpenXR-совместимое PC VR-устройство.
- При использовании PC VR: видеокарта уровня NVIDIA GeForce RTX 2060 или выше, 16 ГБ оперативной памяти, USB 3.0 для кабельного подключения.

*Запуск приложения:*

1. Надеть VR-гарнитуру и убедиться, что контроллеры включены.
2. Запустить приложение из библиотеки (Meta Quest) или из Unity Editor в режиме Play (при разработке).
3. Дождаться загрузки сцены `ShootingRange`. Игрок окажется за столом с оружием.

*Подготовка оружия к стрельбе:*

Таблица 19 — Порядок подготовки оружия к стрельбе

| Шаг | Действие | Результат |
|---|---|---|
| 1 | Нажать кнопку захвата контроллера, направив его на оружие | Оружие прикрепляется к контроллеру |
| 2 | Взять магазин второй рукой | Магазин прикрепляется к свободному контроллеру |
| 3 | Поднести магазин снизу к рукояти оружия | Магазин вставляется в гнездо, щелчок |
| 4 | Взяться свободной рукой за рукоять затвора | Затвор захвачен |
| 5 | Потянуть затвор назад и отпустить | Патрон досылается в патронник |
| 6 | Оружие готово к стрельбе | — |

*Запуск раунда:*

Нажать физическую кнопку на столе. Прозвучит обратный отсчёт 3–2–1, после чего в сцене появится красная граница и начнут появляться мишени. Нажать триггер контроллера для выстрела.

*Перезарядка в ходе раунда:*

1. Извлечь использованный магазин, потянув его из гнезда.
2. Подобрать новый магазин, появившийся на столе (−50 очков).
3. Вставить магазин в гнездо.
4. Передёрнуть затвор для досылания патрона.

*Система очков:*
- Попадание в тело мишени: +50 очков (× мультипликатор дальности).
- Попадание в голову (хэдшот): +100 очков (× мультипликатор дальности).
- Мишень достигла красной границы: −300 очков.
- Подбор нового магазина: −50 очков.
- Мультипликаторы: дальность > 40 м — ×2; дальность > 25 м — ×1,5.

Раунд завершается по истечении таймера или при уходе счёта ниже нуля. После завершения на дисплее отображается итоговая статистика.

**Руководство разработчика**

Данный раздел описывает процедуры расширения и настройки приложения для разработчика, работающего в Unity.

*Добавление нового вида оружия:*

1. В Project-окне Unity вызвать контекстное меню: `Create → Weapons → Config`. Создастся новый файл `WeaponConfig.asset`.
2. Настроить параметры в Инспекторе: `fireMode`, `fireRate`, `range`, `magazineCapacity`, `recoilBack`, `recoilUp`, `casingEjectDirection`.
3. Создать или скопировать префаб оружия. Назначить на него компоненты:

Таблица 20 — Обязательные компоненты префаба оружия

| Компонент | Обязательные ссылки в Инспекторе |
|---|---|

| `AsRifleShoot` / `PistolShoot` | `shootPoint`, `gunanim`, `bang`, `socket`, `bolt` |
| `BoltMechanics` | `cfgjoint`, `trans`, `rb`, `reference`, `GunAnim` |
| `AsRifleGrab` / `PistolGrab` | Все 4–6 GameObject-ссылок на модели рук |

*Изменение параметров существующего оружия:*

Открыть нужный `.asset`-файл в папке проекта (например, `Assets/Weapons/RifleConfig.asset`) и изменить значения в Инспекторе. Изменения применяются ко всем экземплярам оружия без перекомпиляции.

*Настройка HDRP-освещения:*

При добавлении новых статических объектов необходимо пересчитать лайтмапы: `Window → Rendering → Lighting → Generate Lighting`. Для обеспечения корректного освещения динамических объектов (оружие, руки, мишени) убедиться, что в сцене расставлены Light Probe Group.

*Добавление нового типа мишени:*

Создать префаб с коллайдерами, тегом `"Target"` на основном коллайдере и тегом `"TargetHead"` на дочернем (голова). Назначить префаб в соответствующее поле `GameManager`.

Все 14 тест-кейсов пройдены успешно. Составлено руководство по использованию для игрока и разработчика.

## ЗАКЛЮЧЕНИЕ

В ходе выполнения дипломного проекта разработана компьютерная игра с применением XR-технологий — интерактивный стрелковый тренажёр на платформе Unity 6 с поддержкой VR-гарнитур через XR Interaction Toolkit 4.6.0.

По итогам работы достигнуты следующие результаты:

1. Проведён анализ предметной области XR-технологий. Установлено, что мировой рынок AR/VR демонстрирует среднегодовой рост порядка 40 % и к 2026 году, по прогнозам IDC, достигнет 160 млрд долларов. Игровой сегмент является крупнейшим потребителем VR-технологий, а жанр Action/Shooter — наиболее органичным для VR-формата ввиду соответствия физических движений рук игровым действиям.

2. Выполнен сравнительный анализ четырёх аналогов (Half-Life: Alyx, H3VR, Superhot VR, Boneworks). Установлено, что ни одно из рассмотренных приложений не сочетает HDRP-рендеринг, поддержку актуального стандарта OpenXR и систему объективной оценки точности стрельбы. Это подтвердило актуальность разработки.

3. Сформулированы 18 функциональных и 6 нефункциональных требований, охватывающих все аспекты приложения — от механик оружия до производительности в VR.

4. Разработана архитектура приложения на основе компонентной модели Unity. ScriptableObject-конфигурация оружия (`WeaponConfig`) обеспечивает добавление новых видов оружия без изменения кода. Описан конечный автомат GameManager с четырьмя состояниями.

5. Реализована полная система оружия для двух единиц — штурмовой винтовки и пистолета. Система включает: хитскан-стрельбу с разбросом и эффектами (анимация, звук, вспышка, след пробоины), физический затвор на `ConfigurableJoint`, интерактивный магазин через `XRSocketInteractor`, физически симулированные гильзы с рандомизацией, двухфазную отдачу VR-контроллера (`HybridRecoil`), динамическую смену моделей рук при захвате и разрушаемые объекты.

6. Интегрировано XR-взаимодействие с одновременной поддержкой двух платформ (Oculus SDK и OpenXR) через единый XR Management API. Обработка ввода реализована через New Input System с событийной моделью.

7. Разработана игровая логика стрелкового тренажёра: система мишеней с движением и анимацией, многокритериальный подсчёт очков (+50/+100/−300) с мультипликаторами дальности (×2 и ×1,5), управление боезапасом со штрафом, два условия завершения раунда, VFX всплывающих очков и детальная статистика.

8. Проведено тестирование: 14 функциональных тест-кейсов — все со статусом Pass. На устройстве Meta Quest 3 зафиксировано 90 FPS при GPU Frame Time 8–11 мс, что соответствует нефункциональным требованиям.

**Достоинства разработанного приложения:**
- Модульная компонентная архитектура: новое оружие добавляется без изменения существующих скриптов.
- ScriptableObject-конфигурация: мгновенная настройка параметров оружия без перекомпиляции.
- Поддержка двух XR-платформ (Oculus + OpenXR) в одной кодовой базе.
- Физически достоверные механики: ConfigurableJoint-затвор, рандомизированные гильзы, двухфазная отдача контроллера.
- HDRP-рендеринг с запечённым освещением обеспечивает визуальное качество, выходящее за рамки типичных студенческих проектов.

**Недостатки и ограничения:**
- GameManager и система мишеней находятся в стадии реализации; полный цикл раунда не завершён.
- Отсутствуют ИИ-противники и сюжетная составляющая.
- Нет системы сохранения результатов и таблицы рекордов.
- Не реализован сетевой многопользовательский режим.

**Направления дальнейшего развития:**
- Завершение GameManager: полный цикл раунда с динамическим спавном и анимацией мишеней.
- Система рекордов с локальным и облачным сохранением.
- Расширение арсенала: дробовик, снайперская винтовка, пистолет-пулемёт — каждый настраивается через отдельный `WeaponConfig.asset`.
- Оптимизация для Meta Quest 3: LOD-системы, occlusion culling, динамические батчи.
- Сетевой режим для совместной стрельбы двух игроков.

**Трудности при разработке:**

Основной технической сложностью стала настройка `ConfigurableJoint` для физического затвора: добиться корректного ограничения движения строго вдоль одной оси при фиксации двух остальных потребовало глубокого изучения поведения джойнтов в Unity Physics и эмпирического подбора пороговых значений позиции.

Интеграция XR-стека оказалась нетривиальной ввиду смены API: XR Interaction Toolkit 4.6 несовместим с большинством интернет-примеров, написанных под версию 2.x. Потребовалось самостоятельное изучение актуальной документации и примеров пакета.

Составление HDRP Mask Map в GIMP (ручная сборка по цветовым каналам) потребовало понимания физически корректного рендеринга (PBR) и специфики HDRP-материалов.

**Самостоятельно освоенные технологии в ходе проекта:**
- Студент 1: XR Interaction Toolkit 4.6, ConfigurableJoint, HybridRecoil, New Input System, ScriptableObject-паттерн.
- Студент 2: HDRP Mask Map в GIMP, UV-развёртка в Blender 4.x, настройка HDRP Lit-материалов и запечённого освещения.

Дипломный проект в полной мере достиг поставленной цели: разработана компьютерная игра с применением XR-технологий, представляющая собой функциональный стрелковый тренажёр с реалистичными физическими механиками оружия и системой оценки результатов.

```csharp
using System.Collections;
using UnityEngine;
using UnityEngine.XR.Interaction.Toolkit;
using UnityEngine.XR.Interaction.Toolkit.Interactables;
using UnityEngine.XR.Interaction.Toolkit.Interactors;

[RequireComponent(typeof(XRGrabInteractable))]
public class WeaponController : MonoBehaviour
{
    [Header("Config")]
    public WeaponConfig config;

    [Header("References — assign in prefab")]
    public Transform shootPoint;
    public Transform recoilRoot;
    public XRSocketInteractor magazineSocket;
    public Animator weaponAnimator;
    public AudioSource audioSource;
    public AudioClip fireSound;
    public AudioClip emptySound;
    public Light muzzleLight;
    public GameObject impactEffectPrefab;
    public GameObject casingPrefab;
    public Transform casingEjectPoint;

    [Header("Hand Models")]
    public GameObject leftHandPlayer;
    public GameObject rightHandPlayer;
    public GameObject leftHandOnWeapon;
    public GameObject rightHandOnWeapon;
    public GameObject leftHandSecondary;
    public GameObject rightHandSecondary;

    public int CurrentAmmo { get; private set; }
    public bool RoundChambered { get; private set; }
    public bool BoltClosed { get; private set; } = true;

    private bool _triggerHeld;
    private float _nextFireTime;
    private Magazine _magazine;
    private XRGrabInteractable _grab;
    private Vector3 _recoilInitialLocalPos;
    private Vector3 _currentRecoil;
    private Vector3 _targetRecoil;

    public event System.Action<WeaponController> OnGrabbed;
    public event System.Action<WeaponController> OnReleased;

    void Awake()
    {
        _grab = GetComponent<XRGrabInteractable>();
    }

    void Start()
    {
        if (recoilRoot != null)
            _recoilInitialLocalPos = recoilRoot.localPosition;

        SetHandModels(leftHandOnWeapon, false);
        SetHandModels(rightHandOnWeapon, false);
        SetHandModels(leftHandSecondary, false);
        SetHandModels(rightHandSecondary, false);

        _grab.selectEntered.AddListener(OnSelectEntered);
        _grab.selectExited.AddListener(OnSelectExited);

        if (magazineSocket != null)
        {
            magazineSocket.selectEntered.AddListener(OnMagazineInserted);
            magazineSocket.selectExited.AddListener(OnMagazineRemoved);
        }
    }

    void Update()
    {
        UpdateRecoil();

        if (_triggerHeld && Time.time >= _nextFireTime)
        {
            TryFire();
            if (config.fireMode == FireMode.Single)
                _triggerHeld = false;
            _nextFireTime = Time.time + 1f / config.fireRate;
        }
    }

    public void TriggerPress() => _triggerHeld = true;

    public void TriggerRelease()
    {
        _triggerHeld = false;
        weaponAnimator?.Play("Idle");
    }

    public void NotifyBoltOpened() => BoltClosed = false;

    public void NotifyBoltClosed()
    {
        BoltClosed = true;
        ChamberRound();
    }

    public void ChamberRound()
    {
        if (!RoundChambered && CurrentAmmo > 0)
        {
            RoundChambered = true;
            CurrentAmmo--;
            if (_magazine != null) _magazine.Ammo = CurrentAmmo;
        }
    }

    private void TryFire()
    {
        if (!BoltClosed) return;
        if (!RoundChambered)
        {
            PlaySound(emptySound);
            weaponAnimator?.Play("ShootEmpty");
            return;
        }
        Fire();
    }

    private void Fire()
    {
        RoundChambered = false;
        weaponAnimator?.Play("Shoot");
        PlaySound(fireSound);
        ApplyRecoil();
        StartCoroutine(MuzzleFlash());
        StartCoroutine(EjectCasing());
        CastBullet();
        if (config.hasBolt)
            ChamberRound();
    }

    private void CastBullet()
    {
        if (shootPoint == null) return;
        Vector3 dir = shootPoint.forward + new Vector3(
            Random.Range(-0.01f, 0.01f),
            Random.Range(-0.01f, 0.01f), 0f);

        if (!Physics.Raycast(shootPoint.position, dir.normalized,
                             out RaycastHit hit, config.range)) return;
        SpawnImpact(hit);
        if (hit.rigidbody != null)
            hit.rigidbody.AddForceAtPosition(dir * 10f,
                hit.point, ForceMode.Impulse);
        if (hit.collider.CompareTag("Barrel"))
            hit.collider.GetComponent<BarrelExplode>()?.Explode();
    }

    private void SpawnImpact(RaycastHit hit)
    {
        if (impactEffectPrefab == null) return;
        var impact = Instantiate(impactEffectPrefab,
            hit.point + hit.normal * 0.01f,
            Quaternion.LookRotation(hit.normal) * Quaternion.Euler(0, 180, 0));
        impact.transform.localScale = Vector3.one * 0.05f;
        impact.transform.SetParent(hit.collider.transform, true);
        Destroy(impact, 5f);
    }

    private void ApplyRecoil()
    {
        _targetRecoil += new Vector3(0f, config.recoilUp, -config.recoilBack);
    }

    private void UpdateRecoil()
    {
        _targetRecoil = Vector3.Lerp(_targetRecoil, Vector3.zero,
            Time.deltaTime * config.recoilReturnSpeed);
        _currentRecoil = Vector3.Lerp(_currentRecoil, _targetRecoil,
            Time.deltaTime * config.recoilKickSpeed);
        if (recoilRoot != null)
            recoilRoot.localPosition = _recoilInitialLocalPos + _currentRecoil;
    }

    private void OnMagazineInserted(SelectEnterEventArgs args)
    {
        _magazine = args.interactableObject.transform.GetComponent<Magazine>();
        if (_magazine != null) CurrentAmmo = _magazine.Ammo;
    }

    private void OnMagazineRemoved(SelectExitEventArgs args)
    {
        _magazine = null;
        CurrentAmmo = 0;
    }

    private void OnSelectEntered(SelectEnterEventArgs args)
    {
        var t = args.interactorObject.transform;
        bool isLeft  = t.CompareTag("LeftHand");
        bool isRight = t.CompareTag("RightHand");

        if (isLeft)
        {
            bool rightHolding = rightHandOnWeapon != null && rightHandOnWeapon.activeSelf;
            SetHandModels(leftHandPlayer, false);
            SetHandModels(rightHolding ? leftHandSecondary : leftHandOnWeapon, true);
        }
        else if (isRight)
        {
            bool leftHolding = leftHandOnWeapon != null && leftHandOnWeapon.activeSelf;
            SetHandModels(rightHandPlayer, false);
            SetHandModels(leftHolding ? rightHandSecondary : rightHandOnWeapon, true);
        }
        OnGrabbed?.Invoke(this);
    }

    private void OnSelectExited(SelectExitEventArgs args)
    {
        var t = args.interactorObject.transform;
        if (t.CompareTag("LeftHand"))
        {
            SetHandModels(leftHandPlayer, true);
            SetHandModels(leftHandOnWeapon, false);
            SetHandModels(leftHandSecondary, false);
        }
        else if (t.CompareTag("RightHand"))
        {
            SetHandModels(rightHandPlayer, true);
            SetHandModels(rightHandOnWeapon, false);
            SetHandModels(rightHandSecondary, false);
        }
        OnReleased?.Invoke(this);
    }

    private static void SetHandModels(GameObject obj, bool active)
    {
        if (obj != null) obj.SetActive(active);
    }

    private IEnumerator MuzzleFlash()
    {
        if (muzzleLight == null) yield break;
        muzzleLight.enabled = true;
        yield return new WaitForSeconds(0.05f);
        muzzleLight.enabled = false;
    }

    private IEnumerator EjectCasing()
    {
        yield return null;
        if (casingPrefab == null || casingEjectPoint == null) yield break;
        var go = Instantiate(casingPrefab, casingEjectPoint.position,
            casingEjectPoint.rotation * Quaternion.Euler(-90f, 0f, 0f));
        var rb = go.GetComponent<Rigidbody>() ?? go.AddComponent<Rigidbody>();
        Vector3 worldDir = casingEjectPoint.TransformDirection(
            config.casingEjectDirection.normalized);
        rb.AddForce(worldDir * config.casingForce, ForceMode.Impulse);
        Destroy(go, config.casingLifetime);
    }

    private void PlaySound(AudioClip clip)
    {
        if (audioSource != null && clip != null)
            audioSource.PlayOneShot(clip);
    }
}
```

### ПРИЛОЖЕНИЕ Г — HybridRecoil.cs (отдача VR-контроллера)

```csharp
using UnityEngine;
using UnityEngine.XR.Interaction.Toolkit;
using UnityEngine.XR.Interaction.Toolkit.Interactables;

public class HybridRecoil : MonoBehaviour
{
    [Header("Recoil Settings")]
    public float recoilDistance = 0.15f;
    public float recoilAngle    = 8f;
    public float recoilDuration = 0.1f;
    public float returnSpeed    = 5f;

    [Header("References")]
    public XRGrabInteractable grabInteractable;

    private bool isRecoiling = false;
    private float recoilTimer = 0f;

    private class HandData
    {
        public Transform   controller;
        public Vector3     originalPosition;
        public Quaternion  originalRotation;
    }

    private HandData[] hands = new HandData[0];

    void Start()
    {
        if (grabInteractable == null)
            grabInteractable = GetComponent<XRGrabInteractable>();
    }

    public void Fire()
    {
        if (isRecoiling || grabInteractable == null) return;
        var interactors = grabInteractable.interactorsSelecting;
        if (interactors.Count == 0) return;

        hands = new HandData[interactors.Count];
        for (int i = 0; i < interactors.Count; i++)
        {
            var ctrl = interactors[i].transform;
            hands[i] = new HandData {
                controller       = ctrl,
                originalPosition = ctrl.position,
                originalRotation = ctrl.rotation
            };
        }
        isRecoiling = true;
        recoilTimer = 0f;
    }

    void Update()
    {
        if (!isRecoiling || hands.Length == 0) return;
        recoilTimer += Time.deltaTime;

        if (grabInteractable == null ||
            grabInteractable.interactorsSelecting.Count == 0)
        {
            CancelRecoil();
            return;
        }

        for (int i = 0; i < hands.Length; i++)
        {
            if (hands[i]?.controller == null) continue;
            Transform ctrl = hands[i].controller;

            if (recoilTimer <= recoilDuration)
            {
                float t = recoilTimer / recoilDuration;
                Vector3 offset = -ctrl.forward * recoilDistance
                               +  ctrl.up      * (recoilDistance * 0.5f);
                ctrl.position = hands[i].originalPosition + offset * (1f - t);
                ctrl.rotation = hands[i].originalRotation
                              * Quaternion.Euler(-recoilAngle * (1f - t), 0, 0);
            }
            else
            {
                float ret = (recoilTimer - recoilDuration) * returnSpeed;
                if (ret >= 1f)
                {
                    ctrl.position = hands[i].originalPosition;
                    ctrl.rotation = hands[i].originalRotation;
                }
                else
                {
                    Vector3    recoilPos = hands[i].originalPosition
                        - ctrl.forward * recoilDistance
                        + ctrl.up      * (recoilDistance * 0.5f);
                    Quaternion recoilRot = hands[i].originalRotation
                        * Quaternion.Euler(-recoilAngle, 0, 0);
                    ctrl.position = Vector3.Lerp(recoilPos,
                        hands[i].originalPosition, ret);
                    ctrl.rotation = Quaternion.Lerp(recoilRot,
                        hands[i].originalRotation, ret);
                }
            }
        }

        if (recoilTimer > recoilDuration + (1f / returnSpeed))
        {
            isRecoiling = false;
            hands = new HandData[0];
        }
    }

    public void CancelRecoil()
    {
        if (!isRecoiling || hands.Length == 0) return;
        foreach (var h in hands)
        {
            if (h?.controller != null)
            {
                h.controller.position = h.originalPosition;
                h.controller.rotation = h.originalRotation;
            }
        }
        isRecoiling = false;
        hands = new HandData[0];
    }

    void OnDisable() { CancelRecoil(); }
}
```

```csharp
using UnityEngine;

public class CasingSpawner : MonoBehaviour
{
    [SerializeField] private GameObject prefab;
    [SerializeField] private Transform  spawnPoint;
    [SerializeField] private float      impulseForce = 10f;
    [SerializeField] private Vector3    impulseDirection = new Vector3(-1f, 1f, 0f);
    [SerializeField] private float      lifeTime = 60f;

    [SerializeField] private bool    matchSpawnPointRotation = true;
    [SerializeField] private Vector3 rotationOffset = Vector3.zero;

    [SerializeField] [Range(0f, 100f)] private float forceRandomPercent     = 20f;
    [SerializeField] [Range(0f, 100f)] private float directionRandomPercent = 15f;

    public void Spawn()
    {
        if (prefab == null) { Debug.LogError("Префаб не назначен!"); return; }

        Transform target = spawnPoint != null ? spawnPoint : transform;

        Quaternion baseRot  = matchSpawnPointRotation
                              ? target.rotation : Quaternion.identity;
        Quaternion finalRot = baseRot
                              * Quaternion.Euler(rotationOffset)
                              * Quaternion.Euler(-90f, 0f, 0f);

        GameObject obj = Instantiate(prefab, target.position, finalRot);

        Rigidbody rb = obj.GetComponent<Rigidbody>()
                       ?? obj.AddComponent<Rigidbody>();

        Vector3 worldDir = target.TransformDirection(
            RandomizeDirection(impulseDirection).normalized);

        rb.AddForce(worldDir * RandomizeForce(impulseForce), ForceMode.Impulse);
        Destroy(obj, lifeTime);
    }

    public void Spawn(float delay) { Invoke(nameof(Spawn), delay); }

    private float RandomizeForce(float f)
    {
        float p = forceRandomPercent / 100f;
        return f * Random.Range(1f - p, 1f + p);
    }

    private Vector3 RandomizeDirection(Vector3 d)
    {
        float p = directionRandomPercent / 100f;
        return new Vector3(
            d.x * Random.Range(1f - p, 1f + p),
            d.y * Random.Range(1f - p, 1f + p),
            d.z * Random.Range(1f - p, 1f + p)
        ).normalized;
    }

    private void OnValidate()
    {
        impulseDirection = impulseDirection.magnitude > 0
            ? impulseDirection.normalized
            : new Vector3(-1f, 1f, 0f).normalized;
    }
}
```

```csharp
    private XRGrabInteractable grab;

    private void Awake()  { grab = GetComponent<XRGrabInteractable>(); }

    private void Start()
    {
        if (leftHandOnWeapon  != null) leftHandOnWeapon.SetActive(false);
        if (rightHandOnWeapon != null) rightHandOnWeapon.SetActive(false);
    }

    private void OnEnable()
    {
        if (grab == null) return;
        grab.selectEntered.AddListener(OnGrab);
        grab.selectExited.AddListener(OnRelease);
    }

    private void OnDisable()
    {
        if (grab == null) return;
        grab.selectEntered.RemoveListener(OnGrab);
        grab.selectExited.RemoveListener(OnRelease);
    }

    private void OnGrab(SelectEnterEventArgs args)
        => HandleHand(args.interactorObject, false);

    private void OnRelease(SelectExitEventArgs args)
        => HandleHand(args.interactorObject, true);

    private void HandleHand(IXRInteractor interactor, bool showPlayerHand)
    {
        if (interactor == null) return;
        var t = interactor.transform;

        if (t.CompareTag("LeftHand"))
        {
            if (rightHandOnWeapon.activeSelf)
            {
                leftHandModel.SetActive(showPlayerHand);
                leftSecondHand.SetActive(!showPlayerHand);
            }
            else
            {
                leftHandModel.SetActive(showPlayerHand);
                leftHandOnWeapon.SetActive(!showPlayerHand);
            }
        }
        else if (t.CompareTag("RightHand"))
        {
            if (leftHandOnWeapon.activeSelf)
            {
                rightHandModel.SetActive(showPlayerHand);
                rightSecondHand.SetActive(!showPlayerHand);
            }
            else
            {
                rightHandModel.SetActive(showPlayerHand);
                rightHandOnWeapon.SetActive(!showPlayerHand);
            }
        }
    }
}
```
